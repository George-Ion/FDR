False Discovery Rate (FDR) AnalysisThis repository contains computational tools and scripts for managing the False Discovery Rate (FDR) in multiple hypothesis testing. When performing hundreds or thousands of simultaneous tests, the probability of encountering "Type I" errors (false positives) increases dramatically. This project implements methods to control that error rate effectively. Core ConceptsWhat is FDR?The False Discovery Rate is the expected proportion of "discoveries" (rejected null hypotheses) that are actually false positives.$$FDR = E \left[ \frac{V}{R} \right] \text{ where } R > 0$$V: Number of false positives.R: Total number of rejected hypotheses.Methods IncludedBenjamini-Hochberg (BH) Procedure: A step-up procedure that controls the FDR at a specified level $\alpha$. It is more powerful than the family-wise error rate (FWER) controls like the Bonferroni correction.P-Value Distribution: Visualization tools to check if p-values follow a uniform distribution under the null hypothesis.🛠 FeaturesMultiple Testing Correction: Implementations of BH and BY (Benjamini-Yekutieli) procedures.Visualization: Plotting adjusted vs. unadjusted p-values to visualize the "discovery" threshold.Simulation: Scripts to simulate high-dimensional data and observe how FDR behaves under different correlation structures. Getting StartedPrerequisitesEnsure you have the following installed:Python 3.x or R (depending on the specific script used)numpy, scipy, matplotlibUsageLoad your p-values: Prepare a list or array of raw p-values from your statistical tests.Run Correction:Python# Example pseudocode
from statsmodels.stats.multitest import multipletests
rejected, adjusted_p, _, _ = multipletests(pvals, method='fdr_bh', alpha=0.05)
Analyze Results: The output will provide a boolean array of significant results where the FDR is controlled. Why use this over Bonferroni?While the Bonferroni Correction is very safe, it is often too conservative, leading to many "Type II" errors (false negatives).
